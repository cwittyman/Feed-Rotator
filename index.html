Feed Rotator Demo

This demo of Feed Rotator was initialized with the following script:

basicNewsFeed = $('#basic-feed').feedRotator({
	fetchData: true,
	feedUrl: 'http://feeds.feedburner.com/jQueryHowto',
	dataType: 'xml',
	feedContent: 'contentSnippet'
});

The simplest version of the feed can be defined with this call:

$('#basic-feed').feedRotator();

This plugin was developed on jQuery 1.7.1.

A complete list of all available options follows:

  • Defining Your Elements: (Expand)
    • Container Class - The class used by the container of news items. This is set on elements for you, if you pull content from a feed. Otherwise, you should use this class name on the div containing each line of news items.
      (Expected: String) 
      itemClass: '.newsfeed-item'
    • Title Class - The class used by the title content. This is set on elements for you, if you pull content from a feed. Otherwise, you should use this class name on the span containing the news items' titles.
      (Expected: String) 
      titleClass: '.title'
    • Content Class - The class used by the container of news items. This is set on elements for you, if you pull content from a feed. Otherwise, you should use this class name on the div containing each line of news items.
      (Expected: String) 
      contentClass: '.feed-content'
    • Visible Item Class - The class given to the currently visible news item. This is done automatically, but allows for additional styling, if desired.
      (Expected: String) 
      currentItem: '.visible-item'
  • Using a Feed: (Expand)
    • Fetch Data Toggle - This must be set to true to use the provided Feed URL.
      (Expected: Boolean) 
      fetchData: false
    • Feed URL - The URL of the news feed. An XML, JSON, or JSONP feed URL should work.
      (Expected: String) 
      feedUrl: 'http://feeds.feedburner.com/jQueryHowto'
    • Data Type - The type of data to be retrieved from the Feed URL. Should accept one of: 'xml', 'json', 'jsonp', 'script', and 'html'. Currently, 'jsonp' uses $.getJSON() to retrieve the feed, and uses jGFeed otherwise.
      (Expected: String) 
      dataType: 'jsonp'
    • Feed "Title"
      • Static Title Toggle - If set to true, "Title" section is defined by the "staticTitle" option. Otherwise, "Title" is defined from the "feedContent" option.
        (Expected: Boolean) 
        useStaticTitle: false
      • Static Title - String for the "Title" content, if "useStaticTitle" is true.
        (Expected: String) 
        staticTitle: 'Demo Content'
      • Feed Title - The XML Element used to populate the "Title" side of the Feed, if "useStaticTitle" is set to false.
        (Expected: String) 
        feedTitle: 'title'
    • Feed "Content"
      • Feed Content - The XML Element used to populate the "Content" side of the Feed.
        (Expected: String) 
        feedContent: 'summary'
  • Animation Styles: (Expand)
    • Text Line Height - Text line height, used in combination with container height to ensure that only one line of text is visible at a time.
      (Expected: Integer) 
      lineHeight: 20
    • Container Height - The height of the parent "container" that the plugin is initialized on.
      (Expected: Integer) 
      parentHeight: 25
  • Animation Effects: (Expand)
    • Duration - The amount of time it takes for the element to slide out or back in. (By default, it takes 1200ms to slide out, and another 1200ms to slide back in.
      (Unit of Measure: ms, Expected: Integer) 
      duration: 1200
    • Easing - Easing function to use for the animation. (Currently, this has not been tested to see if it works with jQuery UI Easing, but there's no real reason it shouldn't.)
      (Expected: String) 
      slideEasing: 'linear'
    • Animation Overflow - The slides will animate this many pixels outside the container, allowing a small window of time that no slides are visible during the animation time.
      (Unit of Measure: px, Expected: Integer) 
      slideOutOverflow: 40
    • Auto Slide - Should the slider automatically trigger the slide animation periodically? (The auto slide functionality will be paused if a user hovers over the slider container.)
      (Expected: Boolean) 
      autoSlide: false
    • Auto Slide Wait Time - The amount of time after the slide animation stops before the next begins, when Auto Slide is active.
      (Unit of Measure: ms, Expected: Integer) 
      autoSlideTimeout: 4000

And, finally, an explanation of some of the features included in this plugin! (And some extra tricks!)

  • Features: (Expand)
    • News Feed with Rotating Content

      The initial purpose of the plugin was to have a Title and Content section, with the Title staying with its respective Content, while being able to rotate through multiple lines of Content text.

    • Static or Feed-Driven Content

      The requirements initially included only content pulled from a back-end language, such as PHP or C# .NET - which quickly grew to include JSON and XML feeds.

      The plugin is now able to accomodate most content, with a little tweaking of the settings.

    • Looking to the Future....

      Future iterations of this plugin are being looked into, but we don't have many suggestions in line currently. Please feel free to suggest some features!
